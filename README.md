# Picrete Studio — dev.picrete.com

Отдельная платформа, где преподаватели создают и итеративно улучшают ИИ-ассистентов для своих дисциплин
(неорганическая, коллоидная, аналитическая, физическая химия, разделы физики).

## Возможности

- **Провайдеры LLM** — «всеядное» подключение: Yandex AI Studio (YandexGPT 5.x, Alice AI LLM), DeepSeek
  (V4 Pro / V4 Flash), Alibaba Model Studio (Qwen 3.x), любой OpenAI-совместимый API. Ключи хранятся в БД
  зашифрованными (Fernet от `STUDIO_SECRET_KEY`). OpenAI (GPT-5.5) подключается с назначением
  **architect** — используется только для автогенерации системных промптов, в проверке/генерации студентам
  недоступен.
- **Ассистент дисциплины** — профиль (темы, аудитория, описание), критерии оценивания с баллами,
  «нюансы» — экспертные тонкости проверки, которые модель сама не знает.
- **Промпты с версионированием** — системный промпт генерируется моделью-архитектором из профиля по
  best-practices и рекомендациям конкретного семейства (DeepSeek V4: thinking по умолчанию, сэмплинг
  игнорируется; Qwen: RU-принуждение, финальное напоминание; YandexGPT: temperature 0–1, компактность;
  GPT-5.5: без temperature, только reasoning_effort). Версии активируются, сравниваются и откатываются.
- **Генерация типовых заданий** — шаблоны («что варьировать, чего избегать») → банк задач с эталонными
  решениями и рубриками; преподаватель одобряет удачные.
- **Пайплайн проверки** — конструктор: OCR (DataLab) → 1..N LLM-проверщиков → консенсус с порогом
  расхождения и флагом «требует преподавателя».
- **Playground (E2E)** — задача → фото решения → DataLab OCR (редактируемый) → параллельная проверка
  до 6 моделей бок о бок → преподаватель ставит оценки, выбирает «лучшую» и комментирует. Эти вердикты
  копятся в истории — датасет экспертных решений для дальнейшего улучшения промптов и моделей
  (подход Bridgewater/Thinking Machines).

JSON-контракт проверки совместим с прод-пайплайном `Picrete` (`ai_grading.rs`) + добавлены поля
`confidence` и `needs_teacher_review` — отработанные промпты переносятся в основную платформу как есть.

## Структура

```
backend/   FastAPI (Python 3.12), SQLAlchemy async, Postgres (dev: SQLite)
frontend/  React 19 + TS strict + Vite + Tailwind, дизайн-токены «лабораторного журнала»
deploy/    nginx-конфиг для dev.picrete.com
```

## Локальный запуск

```bash
cd backend
python3.12 -m venv .venv && ./.venv/bin/pip install -e .
./.venv/bin/uvicorn app.main:app --port 8100        # SQLite ./data/studio.db, admin/admin

cd ../frontend
npm install && npm run dev                            # http://localhost:5174, proxy /api → 8100
```

## Деплой на сервер (тот же хост, что picrete.com)

1. БД: `CREATE DATABASE picrete_studio OWNER picretesuperuser;`
2. Бэкенд: скопировать репозиторий, создать `.env` рядом с `docker-compose.yml`
   (см. `backend/.env.example`; обязательны `STUDIO_SECRET_KEY`, `STUDIO_FIRST_ADMIN_PASSWORD`,
   `POSTGRES_PASSWORD`, `STUDIO_DATALAB_API_KEY`), затем `docker compose up -d --build`.
   API слушает 127.0.0.1:8100 (network_mode: host).
3. Фронтенд: `cd frontend && npm ci && npm run build`, выложить `dist/` в `/var/www/picrete-studio`.
4. nginx: `deploy/nginx-dev.picrete.com.conf` → `/etc/nginx/sites-enabled/`,
   `certbot --nginx -d dev.picrete.com`, `nginx -s reload`.
5. Зайти на https://dev.picrete.com, сменить пароль администратора, добавить провайдеров и ключи через UI.

## Порядок работы преподавателя

1. «Провайдеры» → подключить Яндекс/DeepSeek/Qwen (+ OpenAI как архитектора).
2. «Ассистенты» → создать дисциплину → заполнить профиль, критерии, нюансы.
3. «Промпты» → сгенерировать v1 архитектором под целевую модель → активировать.
4. «Задания» → шаблоны типовых задач → сгенерировать → одобрить удачные.
5. «Playground» → фото решения → OCR → сравнить модели → отметить лучшую, прокомментировать.
6. Увидели систематическую ошибку → добавить нюанс в профиль → перегенерировать промпт (v2) → снова
   сравнить. Итерации и вердикты сохраняются в истории.
