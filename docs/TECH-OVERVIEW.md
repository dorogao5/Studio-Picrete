**Picrete Studio**

Описание платформы для технических специалистов

**Что это**

Веб-платформа, в которой преподаватель настраивает ИИ-ассистента по
своему курсу. Ассистент генерирует учебные задачи, проверяет работы
студентов и разбирает с ними решения в диалоге. Ответы строятся на
материалах курса, которые загрузил преподаватель: РПД, конспекты лекций,
задачники. Цель: преподаватель настраивает всё сам, без участия
разработчиков.

Адрес: https://dev.picrete.com. Интерфейс рабочий, UX/UI дорабатывается.

**Как проходит настройка**

**1. Материалы.** Преподаватель загружает файлы курса (PDF, Markdown,
TXT, фото страниц). Из PDF текст извлекается напрямую, OCR применяется
только к сканам. Платформа сама разбирает документ и предлагает темы
курса, справочные таблицы, обозначения и формулы. Преподаватель
просматривает список и применяет его одной кнопкой.

**2. Профиль и промпты.** Преподаватель заполняет аудиторию, критерии
оценивания и свои требования к проверке. Системные промпты для трёх
режимов (генерация задач, проверка работ, разбор со студентом) пишет
встроенная модель. Промпты можно править вручную, версии сохраняются.

**3. Задачи.** Преподаватель описывает типовую задачу, при желании
добавляет примеры из задачника. Платформа генерирует задачи партиями.
Каждая задача проверяется автоматически: вторая модель решает её вслепую
и ответы сравниваются, числа из условия сверяются со справочниками
курса, баллы рубрики проверяются на согласованность, дубликаты
отсеиваются. Итог одобряет преподаватель. Одобренные задачи выгружаются
в JSON.

**4. Проверка работ.** Фото рукописной работы распознаётся, проверка
запускается несколько раз и сводится к итоговым баллам по критериям.
Отдельно есть песочница: сравнение ответов разных моделей на одной
задаче и тестовый диалог со студентом.

**Откуда ассистент берёт данные курса**

- справочные таблицы, формулы и обозначения из загруженных материалов
  подставляются в промпт дословно, модель обязана использовать эти
  значения;

- по остальному тексту материалов работает полнотекстовый поиск, в
  промпт попадают только релевантные фрагменты, поэтому размер файлов на
  стоимость запросов не влияет;

- в режиме разбора ассистент не выдаёт готовое решение, а задаёт
  наводящие вопросы и проверяет шаги студента.

**Стек**

|  |  |
|----|----|
| **Компонент** | **Реализация** |
| Backend | Python 3.12, FastAPI, SQLAlchemy (async), Docker |
| Frontend | React 19, TypeScript, Vite, Tailwind, KaTeX для формул |
| База данных | PostgreSQL 16, полнотекстовый поиск tsvector (русская морфология), установлен pgvector |
| Файлы | Yandex Object Storage: оригиналы документов и ночные бэкапы БД |
| Извлечение текста | PyMuPDF; OCR (Datalab) для сканов и фото рукописей |
| LLM | DeepSeek, Qwen, YandexGPT через OpenAI-совместимый клиент; отдельная фоновая модель пишет промпты и разбирает документы |
| Безопасность | JWT, роли администратора и преподавателя, API-ключи провайдеров хранятся в зашифрованном виде |

**Какая модель на каком этапе**

|  |  |  |
|----|----|----|
| **Этап** | **Движок** | **Стоимость** |
| Извлечение текста из PDF | PyMuPDF, локально | бесплатно |
| Индексация и поиск | PostgreSQL, локально | бесплатно |
| Разбор документа | фоновая модель | окна по ~90 тыс. знаков |
| Генерация задач | LLM по выбору преподавателя | партиями |
| Контрольное решение задач | вторая LLM | 1 вызов на задачу |
| Сверка чисел, рубрик, дубликатов | локальные проверки | бесплатно |
| Разбор со студентом, проверка работ | LLM по выбору преподавателя | по запросу |
| OCR | Datalab | за страницу, только для сканов |

**API и экспорт**

- REST API, схема OpenAPI доступна на /docs;

- экспорт одобренных задач в JSON (формат банка задач Picrete) и в
  варианты контрольных;

- результат проверки работы: баллы по критериям, найденные ошибки,
  комментарий студенту, оценка уверенности.

**Что нужно, чтобы завести новую дисциплину**

РПД или регламент курса и конспект либо задачник, в PDF или Markdown.
Дальше около часа работы преподавателя: применить результаты разбора
документов, нажать кнопку сборки промптов, запустить генерацию и
одобрить задачи.
